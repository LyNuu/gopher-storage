# gopher-storage

Сервер для хранения файлов. Бэкенд состоит из двух независимых сервисов, написанных на Go:

- **auth-service** — аутентификация пользователей и управление сессиями.
- **storage-service** — хранение файлов, разграничение доступа и обмен файлами.

## Возможности

- Регистрация и вход пользователей, выдача и обновление JWT-токенов.
- Создание личных и глобальных хранилищ.
- Настройка ограничений хранилища: квота по объёму, максимальный размер файла, список допустимых MIME-типов.
- Загрузка, скачивание и удаление файлов.
- Управление доступом к личному хранилищу через гранты (уровни `read` / `write`).
- Шеринг отдельного файла по временной ссылке без предоставления доступа ко всему хранилищу.

## auth-service

Отвечает за аутентификацию и сессии.

- Регистрация с хешированием пароля (bcrypt).
- Вход: выдаёт пару токенов — access (JWT, HS256) и refresh.
- Refresh-токены хранятся в Redis с TTL и ротируются при обновлении.
- Метаданные пользователей хранятся в PostgreSQL.

Слои: `handler` → `service` → `repository`.

### Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/v1/auth/register` | Регистрация |
| POST | `/api/v1/auth/login` | Вход, возвращает пару токенов |
| POST | `/api/v1/auth/refresh` | Обновление пары токенов |

Порт по умолчанию: `8080`.

## storage-service

Отвечает за жизненный цикл файлов и доступы. Все эндпоинты (кроме скачивания по публичной ссылке) защищены JWT.

- Метаданные хранилищ, файлов, грантов и ссылок — в PostgreSQL.
- Бинарные данные — во внешнем S3-совместимом хранилище (Pydio Cells) через AWS SDK for Go v2.
- Контроль квоты выполняется транзакционно при загрузке.
- Имена файлов валидируются (защита от path traversal).

Слои: `handler` → `service` → `repository` + слой работы с S3 (`s3/pydio`).

### Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/v1/storages` | Создать хранилище |
| GET | `/api/v1/storages` | Список доступных хранилищ |
| GET | `/api/v1/storages/:id` | Получить хранилище |
| POST | `/api/v1/storages/:id/files` | Загрузить файл |
| GET | `/api/v1/storages/:id/files` | Список файлов |
| GET | `/api/v1/storages/:id/files/:name` | Скачать файл |
| DELETE | `/api/v1/storages/:id/files/:name` | Удалить файл |
| POST | `/api/v1/storages/:id/files/:name/share` | Создать временную ссылку на файл |
| POST | `/api/v1/storages/:id/files/:name/upload-link` | Создать presigned-ссылку на загрузку |
| POST | `/api/v1/storages/:id/access` | Выдать доступ пользователю |
| GET | `/api/v1/storages/:id/access` | Список доступов |
| DELETE | `/api/v1/storages/:id/access/:userId` | Отозвать доступ |
| DELETE | `/api/v1/shares/:token` | Отозвать ссылку |
| GET | `/api/v1/shared/:token` | Скачать файл по ссылке (без авторизации) |

Порт по умолчанию: `8082`.

## Технологии

- Go, Echo
- PostgreSQL (pgx)
- Redis
- S3-совместимое хранилище Pydio Cells (AWS SDK for Go v2)
- JWT (golang-jwt), bcrypt
- go-playground/validator

## Деплой
Бэкенд развёрнут на сервере. Настроен CI/CD через GitHub Actions: при пуше в репозиторий выполняется автоматическая сборка и деплой на сервер.
